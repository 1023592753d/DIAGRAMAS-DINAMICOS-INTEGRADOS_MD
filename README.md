# DIAGRAMAS-DINAMICOS-INTEGRADOS_MD

**Sistema de Gestión de Proyectos de Grado – FESC**

 **INTRODUCCION**

Este portafolio contiene la documentación UML dinámica completa para el sistema integral de gestión de proyectos de grado de la FESC.
El objetivo es modelar todas las interacciones, procesos y transiciones del ciclo de vida del proyecto de grado, desde su propuesta inicial hasta la aprobación final.


 **Contenido del Portafolio**

El documento consolidado incluye:

 **1. Diagrama de casos de uso completo**

* Actores: Estudiante, Tutor, Jurado, Coordinador, Sistema.
* Casos de uso principales y secundarios.
* Relaciones *include* y *extend*.

 **2. Especificaciones detalladas de casos de uso críticos**

* Caso crítico modelado: **Sustentar Proyecto**
* Precondiciones, flujo principal, flujos alternativos y postcondiciones.

 **3. Diagramas de secuencia**

* Secuencia principal del proceso de sustentación
* Participación de todos los actores del sistema
* Mensajes y validaciones incluidas.

 **4. Diagramas de comunicación**

* Modelo estructural del flujo de sustentación
* Objetos, enlaces, y mensajes numerados.

 **5. Diagrama de estados**

* Estados completos del proyecto de grado:
  Propuesta → En Revisión → Desarrollo → Sustentación → Calificación → Finalizado.

 **6. Matriz de trazabilidad**

* Requisitos del sistema
* Casos de uso asociados
* Cobertura del 100%.

**7. Documento de integración**

* Explicación de cómo se relacionan entre sí:

  * Casos de uso
  * Secuencias
  * Comunicación
  * Estados

 **8. Análisis de cobertura**

* Verificación de que todos los requisitos están representados en los diagramas.

 **9. Identificación de puntos críticos**

* Riesgos detectados y estrategias de mitigación.
